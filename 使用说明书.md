# typetype Windows 使用说明书

## 当前使用版本

- 程序位置：`C:\Users\51696\Desktop\typetype\typetype.exe`
- 日志位置：`C:\Users\51696\Desktop\typetype\typetype.log`
- 当前已配置：
  - 识别模式：流式输出
  - 翻译目标：粤语
  - LLM 润写：MiniMax-M2.7 兼容模式
  - 开机自动启动：开启

## 快捷键

- `Ctrl + /`：语音输入原文。
- `Ctrl + .`：语音翻译。

按一次开始录音，再按一次结束录音。

## 1. 流式原文输入测试

1. 打开记事本、浏览器输入框、微信输入框等可输入文字的位置。
2. 点击输入框，确保光标在输入框内。
3. 按 `Ctrl + /` 开始说话。
4. 正常情况下，流式模式会边说边输出原文。
5. 再按 `Ctrl + /` 停止录音。

本次修复后，Windows 流式音频 chunk 已统一降采样到 16kHz，避免“有录音 chunk 但识别不出原文”的问题。

流式原文输入会根据停顿自动断句：继续说下一段时自动补中文逗号，停止录音时自动补句号或问号。

## 2. 粤语翻译测试

1. 点击目标输入框。
2. 按 `Ctrl + .` 开始录音。
3. 说一段普通话。
4. 再按 `Ctrl + .` 停止录音。
5. 程序会先用非流式 SenseVoice 识别整段语音，再翻译成粤语并粘贴。

说明：当前翻译不是逐字流式翻译。翻译模型需要拿到完整文本后再翻译，所以翻译请使用 `Ctrl + .`。在流式输出模式下，`Ctrl + .` 仍然可用，会自动切到翻译链路。

## 3. LLM 结构化润写测试

LLM 润写适合在非流式语音输入中测试，因为流式输入优先保证原文实时输出。

1. 打开设置，把识别模式切换为“非流式”。
2. 确认“启用 LLM 润写”已开启。
3. 提供商选择“国产/第三方模型（Base URL 自定义）”。
4. 按下表填写 Base URL 和模型。
5. 填入客户自己的 API Key。
6. 点击“测试连接”，成功后再保存。
7. 推荐先用 MiniMax-M2.7 测试：
   - Base URL：`https://api.minimaxi.com/v1`
   - 模型：`MiniMax-M2.7`

### 小白填写步骤

1. “提供商”选择：`国产/第三方模型（Base URL 自定义）`。
2. “Base URL”复制上表对应地址，不要在后面手动加 `/chat/completions`。
3. “API Key”填客户在该厂商控制台创建的密钥。
4. “模型”填上表推荐模型，或填厂商控制台显示的模型 ID。
5. 点击“测试连接”。
6. 如果失败，优先检查三件事：Base URL 是否复制完整、API Key 是否属于同一个厂商、模型名是否在账号中可用。

### 结构化润写效果

1. 对输入框按 `Ctrl + /`，说一段较长内容，例如：

   今天开会主要有三件事第一产品这周要把流式输入修好第二翻译功能粤语要继续测试第三我下周一之前整理使用说明另外风险是开机自启动可能在绿色版里注册表名称不稳定需要修复

2. 再按 `Ctrl + /` 停止。

预期效果：短句会自动补标点；长段会自动梳理跳跃表达、修正明显口误或笔误，在不遗漏关键信息、数据和结论的前提下，整理为段落、清单、事项、风险、待办或“一、→（一）→1.”这类层级文档，接近 Typeless 的结构化体验。

## 4. 开机自动启动测试

设置里开启“开机自动启动”后，Windows 注册表会写入：

`HKCU\Software\Microsoft\Windows\CurrentVersion\Run\typetype`

值应指向：

`C:\Users\51696\Desktop\typetype\typetype.exe --launch-at-login`

本次修复后，旧的 `electron.app.Electron` 和临时目录启动项会被清理。

## 5. 排查方法

- 如果没有粘贴文字，先确认目标输入框有光标。
- 如果快捷键没反应，打开设置查看快捷键是否注册成功。
- 如果翻译首次很慢，是翻译模型加载或缓存导致，后续会快一些。
- 如果 LLM 润写失败，检查是否选择“国产/第三方模型”，Base URL、API Key、模型名是否来自同一家厂商。
- 详细错误请查看 `C:\Users\51696\Desktop\typetype\typetype.log`。
